\subsection{Sequence Modelling}
Names are relatively short sequences of characters whose internal order carries meaningful phonetic, morphological and cultural coherence, making them natural candidates for sequential modelling. Before the advent of neural approaches, sequence generation was dominated by Markov chain models and $n$-gram language models, which capture only local character dependencies and struggle to model longer-range phonetic and morphological structure \cite{jurafsky_speech_2026}. Later, even feedforward neural networks proved to be ill-suited for this task, as they require fixed-size inputs and outputs \cite{sutskever_sequence_2014}, making them inappropriate for variable-length sequences such as words. Recurrent Neural Networks, introduced by Elman \cite{elman_finding_1990}, were specifically designed to address these limitations and, as such, dominated sequence modelling for two decades. More recently, transformer-based architectures \cite{vaswani_attention_2017} have achieved state-of-the-art results across many sequence modelling tasks. However, for short sequences such as names, the self-attention mechanism can introduce unnecessary complexities compared to recurrent approaches, which remain a natural fit for character-level generation.\\

\subsection{Recurrent Neural Networks}
While traditional feedforward neural networks typically represent time as an additional dimension of the input, a Recurrent Neural Network (RNN) represents time implicitly through its effects on processing, by incorporating recurrent connections that give the network a form of dynamic memory \cite{elman_finding_1990}. When applied to a sequential task, this formulation is autoregressive: the sequence is generated one character at a time, with each character conditioned on all preceding characters \cite{graves_generating_2013, elman_finding_1990}. At each timestep, the hidden state, which acts as context, is updated as a function of the current learned character embedding and of the previous hidden state, with the recurrent connection acting as an implicit memory that carries forward a summary of the sequence seen so far. For a simple RNN with one hidden layer, the hidden layer activation $h_t$ and the output $y_t$ are computed by iterating the following equation from $t = 1$ to $T$:
\begin{align}
    h_t &= \mathcal{H}(W_{xh}\, x_t + W_{hh}\, h_{t-1} + b_h) \\
    y_t &= \mathcal{Y}(W_{hy}\, h_t + b_y)
\end{align}

where $x_t$ is the input, $W_{xh}$ and $W_{hy}$ are weights and $b_h$ and $b_y$ are bias terms, while $\mathcal{H}$ denotes the hidden layer activation function and $\mathcal{Y}$ denotes the output layer activation function. The probability given by the network to the input sequence $\mathbf{x}$ is:
\begin{equation}
    \Pr(x) = \prod_{t=1}^{T} \Pr(x_{t+1} \mid y_t)
\end{equation}

The network is trained by minimising the negative log-likelihood of the correct output sequence:
\begin{equation}
\mathcal{L} = -\sum_{t=1}^{T} \log \Pr(x_{t+1} \mid y_t)
\end{equation}

Despite their simplicity and effectiveness on short sequences, Vanilla RNNs have been found to suffer from the vanishing or exploding gradient problem, whereby gradients tend to vanish or explode as they are backpropagated through time, making it difficult to capture long-range dependencies within sequences \cite{bengio_learning_1994}. As a result, Long Short-Term Memory (LSTM) networks, introduced by Hochreiter and Schmidhuber \cite{hochreiter_long_1997} and Gated Recurrent Unit (GRU) networks, introduced by Cho et al. \cite{cho_learning_2014}, became increasingly popular alternatives to vanilla RNNs, as both architectures introduce gating mechanisms that better control the flow of information through the network, mitigating the vanishing gradient problem and enabling the model to better capture long-range dependencies in the data.\\

Similarly to vanilla RNNs, LSTMs feature a repeated module or memory cell, as depicted in Fig. 3.1. In LSTMs, however, this module consists of three gating mechanisms, a "forget" gate, an "input" gate and an "output" gate. To the hidden state that is typical of RNNs, LSTMs add a cell state, which is responsible for long-term memory, leaving the hidden state responsible of short-term memory.\\

    \centering
    \includegraphics[width=0.6\textwidth]{images/LSTM.png}
    \caption{Long Short-term Memory Cell. Image taken from \cite{graves_generating_2013}.}
    \label{fig:lstm}
\end{figure}

At each timestep, the network updates the cell state $c_t$ by deciding, based on the input $x_t$ and on the previous hidden state $h_{t-1}$, what to forget of the previous cell state $c_{t-1}$ via the forget gate $f_t$ and how much of the new candidate information to write into the cell state via the input gate $i_t$. The network also updates the hidden state $h_t$ by filtering the cell state $c_t$ through the output gate $o_t$, which controls how much of the long-term memory is exposed as the current hidden state. The LSTM hidden layer function $\mathcal{H}$ is implemented by the following equations \cite{graves_generating_2013}:\\
\begin{align}
    i_t &= \sigma(W_{xi}x_t + W_{hi}h_{t-1} + W_{ci}c_{t-1} + b_i) \\
    f_t &= \sigma(W_{xf}x_t + W_{hf}h_{t-1} + W_{cf}c_{t-1} + b_f) \\
    c_t &= f_t c_{t-1} + i_t \tanh(W_{xc}x_t + W_{hc}h_{t-1} + b_c) \\
    o_t &= \sigma(W_{xo}x_t + W_{ho}h_{t-1} + W_{co}c_t + b_o) \\
    h_t &= o_t \tanh(c_t)
\end{align}

In deeper architectures with multiple hidden layers, skip connections from the input to all hidden layers and from all hidden layers to the output are often employed \cite{graves_generating_2013}, reducing the number of processing steps between the bottom and top of the network and further mitigating the vanishing gradient problem.\\

As shown by Graves \cite{graves_generating_2013}, LSTMs can be used to generate sequences of characters autoregressively, demonstrating that recurrent architectures are well suited to discrete sequential data such as text. In \cite{graves_generating_2013}, an input vector sequence $\mathbf{x} = (x_1, \ldots , x_T)$, where $x_t$ is a character embedding, is passed through weighted connections to a stack of N recurrently connected hidden layers to compute first the hidden vector sequences $\mathbf{h} = (h_1, \ldots , h_T )$ and then the output vector sequence $\mathbf{y} = (y_1, \ldots , y_T )$. Each output vector $y_t$ is used to parameterise a predictive distribution $\text{Pr}(x_{t+1}\mid y_t)$ over the possible next inputs $x_{t+1}$. This is achieved by using a softmax activation function at the output layer.\\

During training, \textit{teacher forcing} is often used to train the weights of an RNN. This involves feeding the embedding of the ground-truth character back into the model at each step, thus forcing the RNN to stay close to the ground-truth sequence. At inference time, assuming that predictions are probabilistic, a common approach involves \textit{greedy decoding}, whereby the character with the highest probability at time $t$ is picked and fed back into the network as input at time $t+1$. An interesting alternative used in \cite{graves_generating_2013} consists of iteratively sampling from the network's output distribution and feeding the sample back as input at the next timestep. While the network itself remains deterministic, this sampling step introduces stochasticity into the model. When RNNs are used as building blocks of generative models, this stochasticity becomes a strength, allowing the model to produce diverse and novel outputs. Another common appraoch in in Sequence-to-sequence (Seq2Seq) tasks is \textit{beam search}. At each timestep, beam search expands all possible continuations of the current partial sequences, ranks them according to their cumulative log-probability under the model and retains only the top $k$ candidates, where $k$, the beam size, is a hyperparameter. This allows the model to explore multiple hypotheses simultaneously rather than committing to a single path.\\

\subsection{Autoencoders}
Autoencoders constitute a type of neural network architecture designed to efficiently compress (or encode) input data, then reconstruct (or decode) the original input from a compressed representation, also known as \textit{code} or \textit{latent representation}. During training, autoencoders are typically optimised via backpropagation to minimise reconstruction loss, i.e., the discrepancy between the reconstructed data and the original input, in a self-supervised fashion. This pushes the model to learn a compact and meaningful latent representation. Autoencoders are typically composed of two parametrised functions: an \textit{encoder} $E_\phi:\mathcal{X}\rightarrow\mathcal{Z}$, parametrised by $\phi$ and a \textit{decoder}, $D_\theta:\mathcal{Z}\rightarrow\mathcal{Y}$, parametrised by $\theta$. The encoder comprises layers that yield an increasingly compressed representation of the input data through dimensionality reduction. Between encoder and decoder is a \textit{bottleneck}, which contains the most compact representation of the input: this is both the output layer of the encoder network and the input layer of the decoder network. The decoder decompresses the encoded representation of data, ultimately reconstructing the original input. For any input $\mathbf{x} \in \mathcal{X}$, its latent representation is $\mathbf{z} = E_\phi(\mathbf{x})$ and the decoded representation is $\mathbf{x}'= D_\theta(\mathbf{z}) = D_\theta(E_\phi(\mathbf{x}))$.\\

When the RNN is used as a building block for an autoencoder in a sequence generation task, the architecture is typically referred to as a Recurrent Autoencoder (RAE) or Seq2Seq model \cite{sutskever_sequence_2014}. Here, both the encoder and decoder are RNNs, commonly implemented using LSTMs. The encoder processes the input sequence token by token - here, character by character - and compresses it into a fixed-dimensional vector representation, while the decoder generates the output sequence autoregressively, one token at a time. In an RNN-based autoencoder, there is no progressive dimensionality reduction through layers, since all hidden layers maintain the same dimensionality throughout. At $t=1$, the encoder receives a start-of-sequence embedding as input and initialises its hidden state (and cell state, in the case of an LSTM) to zero vectors. At each subsequent timestep, the first recurrent layer updates its hidden state using the current input embedding and its previous hidden state, while higher layers receive as input the hidden state of the layer below at the same timestep and maintain their own recurrent dynamics across timesteps. Although the encoder produces hidden states at every timestep, only the final hidden state of the top layer after processing the final input token is retained. This fixed-dimensional vector constitutes the latent representation and must encode all relevant information about the variable-length input sequence. The code is used to initialise the hidden state of the decoder, which reconstructs the sequence autoregressively, one character at a time, conditioned on the latent vector and on the previously generated character embedding. At training time, teacher forcing is typically employed to train the decoder. At inference time, the decoder generates characters greedily, via sampling from the output distribution or via beam search, until an end-of-sequence token is produced.\\

In Seq2Seq tasks, the encoder is often implemented as a Bidirectional RNN (BRNN) \cite{schuster_bidirectional_1997}. A BRNN consists of two recurrent networks: one processes the input sequence in the forward temporal direction, while the other processes the sequence in reverse order. The hidden states from both directions are then combined, enabling the encoder to capture both past and future contextual information for each timestep.\\

In discrete tasks such as text generation, cross-entropy loss is typically used to compute the reconstruction loss. Minimising cross-entropy loss at each timestep is equivalent to maximising the conditional log-likelihood of the correct output sequence given the latent representation $\mathbf{z}$ and the previous ground-truth characters, if using teacher forcing, thereby encouraging the decoder to faithfully reconstruct the original input sequence. At each timestep $t$, the decoder outputs a probability distribution over the vocabulary of possible next characters via a softmax activation function. The cross-entropy loss measures the discrepancy between this predicted distribution and the true next character, represented as a one-hot vector. The total reconstruction loss over an output sequence of length $T$ is therefore given by:
\begin{equation}
\mathcal{L}_{\text{reconstruction}} = -\sum_{t=1}^{T} \log p_\theta(y_t \mid y_{1}, \ldots, y_{t-1}, \mathbf{z})
\end{equation}

where $\mathbf{z}$ is the latent vector produced by the encoder and $y_t$ is the true character at timestep $t$. Via backpropagation, the model parameters $\phi*$ and $\theta*$ that minimise the cross-entropy loss are learned during training. Since the output sequence includes start and end of sequence tokens, the total sequence length for a name of length $L$ is $T=L+2$.

\subsection{Variational Autoencoders}
Variational Autoencoders (VAEs) are a class of latent variable generative models \cite{kingma_auto-encoding_2022}. In contrast to discriminative models, which learn a conditional distribution $p(\mathbf{y} \mid \mathbf{x})$ mapping inputs to labels, generative models learn the underlying distribution of the data $p(\mathbf{x})$ itself,from which new samples resembling the observed data can be generated. As shown in \cite{bowman_generating_2016} in the context of sentence generation, standard autoencoders do not learn a smooth and dense latent space but one that typically exhibits discontinuities or \textit{holes}, i.e., regions that do not correspond to any training example and from which decoding produces incoherent outputs. As a result, following a path or \textit{homotopy} between the encodings for two sentences, the intermediate sentences generally appear to be ungrammatical and do not transition smoothly from one to the other. VAEs are based on a regularised version of the standard autoencoder, meaning that they enforce a prior distribution or geometry over the latent space. As a result, the model can learn a continuous latent space, from which new, proper samples can be drawn.\\

We assume that the observed variable $\mathbf{x}$ is drawn from an unknown true distribution $p^{*}(\mathbf{x})$. Since the true data generating process is unknown, this distribution is approximated using a parametric model $p_\theta(\mathbf{x})$ with parameters $\theta$, where the optimal parameters $\theta^*$ are such that they maximise the log-likelihood of the observed dataset:

\begin{equation}
\theta^{*} = \arg\max_{\theta} \prod_{i=1}^{n} p_\theta(\mathbf{x}^{(i)}) =\arg\max_{\theta} \sum_{i=1}^{n} \log p_\theta(\mathbf{x}^{(i)})
\end{equation}

where $n$ is the number of observations. VAEs introduce the latent variable $\mathbf{z}$ and define a joint distribution over both the observed variables $\mathbf{x}$ and the latent variables $\mathbf{z}$ as follows:
\begin{equation}
p_\theta(\mathbf{x}, \mathbf{z}) = p_\theta(\mathbf{x} \mid \mathbf{z}) p(\mathbf{z})
\end{equation}

where \(p_\theta(\mathbf{x} \mid \mathbf{z})\) is the \textit{likelihood} and $p(\mathbf{z})$ is the prior distribution over $\mathbf{z}$ or, simply, the \textit{prior}, since it is not conditioned on any observations. The prior $p(\mathbf{z})$ is often chosen to be a standard multivariate Gaussian $\mathcal{N}(\boldsymbol{0}, \mathbf{I})$ or, in other words, a \textit{hypersphere} centred at the origin in the latent space. The \textit{marginal likelihood} or \textit{model evidence} is:
\begin{equation}
p_\theta(\mathbf{x}) = \int p_\theta(\mathbf{x} \mid \mathbf{z}) p(\mathbf{z}) d\mathbf{z} = \int p_\theta(\mathbf{x}, \mathbf{z}) d\mathbf{z}
\end{equation}

which is generally intractable, as it requires integrating over all possible values of the latent variable $\mathbf{z}$. Its intractability is related to the intractability of the posterior distribution over latent variables:
\begin{equation}
p_\theta(\mathbf{z} \mid \mathbf{x})
=
\frac{p_\theta(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z})}
{p_\theta(\mathbf{x})}
\end{equation}

Kingma and Welling \cite{kingma_auto-encoding_2022} address this intractability by introducing $q_\phi(\mathbf{z} \mid \mathbf{x})$ whose parameters $\phi$ can be optimised such that:
\begin{equation}
q_\phi(\mathbf{z} \mid \mathbf{x}) \approx p_\theta(\mathbf{z} \mid \mathbf{x})
\end{equation}

As shown in Fig. 3.2, a VAE can be viewed as two coupled, but independently parameterised models: the encoder or recognition model $q_\phi(\mathbf{z} \mid \mathbf{x})$ and the decoder or generative model $p_\theta(\mathbf{x} \mid \mathbf{z})$, where $\phi$ denotes the parameters of the recognition model, while $\theta$ denotes the parameters of the generative model.\\
\begin{figure}[h]
    \centering
    \includegraphics[width=0.6\textwidth]{images/VAE.png}
    \caption{Variational Autoencoder. Image taken from \cite{kingma_introduction_2019}.}
    \label{fig:vae}
\end{figure}

For any choice of inference model $q_\phi(\mathbf{z} \mid \mathbf{x})$, the log-likelihood of the data is:
\begin{align}
\log p_\theta(\mathbf{x}) 
&= \underbrace{\mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})} \left[ \log \frac{p_\theta(\mathbf{x}, \mathbf{z})}{q_\phi(\mathbf{z}|\mathbf{x})} \right]}_{\mathcal{L}_{\theta,\phi}(\mathbf{x}) \ (\text{ELBO})}
+ \underbrace{\mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})} \left[ \log \frac{q_\phi(\mathbf{z}|\mathbf{x})}{p_\theta(\mathbf{z}|\mathbf{x})} \right]}_{D_{\text{KL}}(q_\phi(\mathbf{z}|\mathbf{x}) \| p_\theta(\mathbf{z}|\mathbf{x}))}
\end{align}

where the first term is the variational lower bound, often called the evidence lower bound (ELBO) and the second term is the Kullback-Leibler (KL) divergence between $q_\phi(\mathbf{z}|\mathbf{x})$ and $p_\theta(\mathbf{z}|\mathbf{x})$. Since the KL divergence is non-negative, the ELBO constitutes a lower bound on the log-likelihood of the data.\\

Maximizating the ELBO $\mathcal{L}_{\theta,\phi}(\mathbf{x})$ w.r.t. the parameters $\theta$ and $\phi$ will concurrently approximately maximise the marginal likelihood $p_\theta(\mathbf{x})$, leading to a better generative model, and minimise the KL divergence of the approximation $q_\phi(\mathbf{z} \mid \mathbf{x})$ from the true posterior $p_\theta(\mathbf{z} \mid \mathbf{x})$, leading to a better $q_\phi(\mathbf{z} \mid \mathbf{x})$. As a result, the VAE uses the ELBO as its maximisation objective. Rearranging equation (3.16) and expanding the joint distribution $p_\theta(\mathbf{x},\mathbf{z})=p_\theta(\mathbf{x} \mid \mathbf{z})p(\mathbf{z})$ yields:
\begin{equation}
\log p_\theta(\mathbf{x}) - D_{\text{KL}}(q_\phi(\mathbf{z}|\mathbf{x}) \| p_\theta(\mathbf{z}|\mathbf{x})) = \mathbb{E}_{\mathbf{z} \sim q_\phi(\mathbf{z}|\mathbf{x})} \log p_\theta(\mathbf{x}|\mathbf{z}) - D_{\text{KL}}(q_\phi(\mathbf{z}|\mathbf{x}) \| p(\mathbf{z}))
\end{equation}

The negation of the above defines the loss function of VAE as:
\begin{align}
\mathcal{L}_{\text{VAE}}(\theta, \phi) &= -\log p_\theta(\mathbf{x}) + D_{\text{KL}}(q_\phi(\mathbf{z}|\mathbf{x}) \| p_\theta(\mathbf{z}|\mathbf{x})) \\
&= -\mathbb{E}_{\mathbf{z} \sim q_\phi(\mathbf{z}|\mathbf{x})} \log p_\theta(\mathbf{x}|\mathbf{z}) + D_{\text{KL}}(q_\phi(\mathbf{z}|\mathbf{x}) \| p(\mathbf{z})) \\
\theta^{*}, \phi^{*} &= \arg\min_{\theta, \phi} \mathcal{L}_{\text{VAE}}
\end{align}

which consists of two terms, (i) reconstruction and (ii) KL regularization.\\

$\mathcal{L}_{\text{VAE}}(\theta, \phi)$ is known as the \textit{variational lower bound} or \textit{evidence lower bound}: since the KL divergence is always non-negative, $-\mathcal{L}_{\text{VAE}}(\theta, \phi)$ is the lower bound of the marginal log-likelihood $\log p_\theta(\mathbf{x})$.\\

The expectation term in the loss function involves sampling from the approximate posterior distribution $\mathbf{z} \sim q_\phi(\mathbf{z} \mid \mathbf{x})$. Since sampling is a stochastic operation, it is not directly differentiable, which prevents gradients from flowing through it during backpropagation. To address this issue, Kingma and Welling \cite{kingma_auto-encoding_2022} propose a \textit{reparameterisation trick}, which expresses the latent variable $\mathbf{z}$ as a deterministic function of the encoder outputs and of an auxiliary noise variable $\boldsymbol{\epsilon}$:
\begin{align}
\mathbf{z} &= \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon}, 
\quad \text{where } \boldsymbol{\epsilon} \sim \mathcal{N}(\boldsymbol{0}, \mathbf{I}),
\end{align}
where $\odot$ denotes element-wise multiplication. The noise variable $\boldsymbol{\epsilon} \sim \mathcal{N}(\boldsymbol{0}, \mathbf{I})$ is independent of the model parameters $\phi$ and all stochasticity is delegated to it, enabling gradient-based optimisation of the encoder and decoder parameters. A common choice for the approximate posterior is a multivariate Gaussian with diagonal covariance, parameterised by the encoder network, which outputs the mean and standard deviation vectors $\boldsymbol{\mu}_\phi(\mathbf{x})$ and $\boldsymbol{\sigma}_\phi(\mathbf{x})$:
\begin{align}
q_\phi(\mathbf{z}|\mathbf{x}) &= \mathcal{N}\!\big(\mathbf{z}; \boldsymbol{\mu}_\phi(\mathbf{x}), \mathrm{diag}(\boldsymbol{\sigma}^2_\phi(\mathbf{x}))\big).
\end{align}

For continuous latent variables and a differentiable encoder and generative model, the ELBO can be straightforwardly differentiated w.r.t. both $\phi$ and $\theta$ thanks to this reparameterisation trick. This allows joint optimization w.r.t. all parameters ($\phi$ and $\theta$) using stochastic gradient descent (SGD).\\

\subsection{Posterior collapse, $\beta$-VAE and homotopy}
Variational autoencoders quickly gained great popularity in continuous domains, however \cite{bowman_generating_2016} show that they struggle with discrete tasks such as word or sentence modelling. A model that encodes useful information in the latent variable $\mathbf{z}$ should typically exhibit a non-zero KL divergence and a relatively small cross-entropy term. If the KL divergence is zero, the latent representation $z$ becomes uninformative and the model yields the same $\mu$ and $\sigma$ for every input. Especially with discrete tasks, which involve the use of an autoregressive decoder, VAEs during training often learn to ignore the code $\mathbf{z}$: early in training, the encoder hasn't learned meaningful representations and, since the KL term forces the posterior to match the prior, the model finds it easier to set \(q(\mathbf{z}|\mathbf{x}) = p(\mathbf{z})\) or, in other words, to drop the KL penalty entirely rather than learning to extract useful features from the input. Perhaps more intuitively, a powerful-enough decoder might not need the latent representation $\mathbf{z}$ at all to (re-)construct the sentence autoregressively with little reconstruction error as all it needs to predict a token might be the input, i.e., the previous token. As a result, a vanilla VAE, when used for a discrete task like text generation, tends to collapse to a simple RNN. This is often referred to as \textit{posterior collapse}.\\

VAEs suffer from another failure mode. While they can succeed in learning a compressed, probabilistic representation of high-dimensional data, this representation is often not very interpretable \cite{higgins_-vae_2017}. Better interpretability can be achieved by aiming for a more disentangled representation, defined as one where single latent units are sensitive to changes in single generative factors, while being relatively invariant to changes in other factors \cite{bengio_representation_2014}.\\

To overcome the limitations above, the $\beta$-VAE, introduced by Higgins et al. \cite{higgins_-vae_2017}, extends the standard VAE by introducing a hyperparameter $\beta$ into the loss function:
\begin{equation}
\mathcal{L}_{\beta\text{-VAE}}(\theta, \phi, \beta) = -\log p_\theta(\mathbf{x}) + \beta D_{\text{KL}}(q_\phi(\mathbf{z}|\mathbf{x}) \| p_\theta(\mathbf{z}|\mathbf{x}))
\end{equation}

While $\beta = 1$ corresponds to the original formulation by \cite{kingma_auto-encoding_2022}, a $\beta > 1$ puts a stronger constraint on the latent space, encouraging the model to learn disentangled and interpretable representations of the conditionally independent data generative factors.\\ 

During training, \cite{bowman_generating_2016} proposes to initially set $\beta$ to zero, so that the model is incentivised to encode as much information in $\mathbf{z}$ as it can. As training progresses, $\beta$ is gradually increased, forcing the model to smooth out its encodings and pack them into the prior. The rate of increase is treated as a hyperparameter. As a result, KL spikes early in training while the model can encode information in $\mathbf{z}$ cheaply, then drops substantially once it begins paying the full KL divergence penalty and finally slowly rises again before converging as the model learns to condense more information into $\mathbf{z}$. Fu et al. \cite{fu_cyclical_2019} refer to this as a \textit{monotonic annealing schedule} and propose, instead, a \textit{cyclical annealing schedule}, whereby the process of increasing $\beta$ is repeated multiple times. By reopening the path at $\beta = 0$, the cyclical schedule progressively improves the performance, by leveraging good latent codes learned in the previous cycles as warm re-starts. \cite{fu_cyclical_2019} also shows that a cyclical schedule leads to a more structured latent representation compared to the monotonic annealing scheduling, which can be beneficial in downstream applications.\\

Another remedy to posterior collapse is to weaken the decoder via a variant of word dropout \cite{bowman_generating_2016}. A natural way to weaken the decoder is to remove some or all of the information that it uses during learning by randomly replacing some fraction of the conditioned-on word tokens with a generic unknown word token. Forcing the model to at times rely solely on the latent variable $\mathbf{z}$ to make good predictions incentivises it to learn a meaningful representation.\\

Bowman et al. \cite{bowman_generating_2016} explore the concept of \textit{homotopy} within sentence generation. The smooth and dense nature of the latent space of a variational autoencoder is such that it allows to interpolate between codes to generate a latent representation which can then be passed on to the decoder. In this context, a homotopy between two codes $z_1$ and $z_2$ is the set of points on the line between them, inclusive of those, $z(t) = z_1(1−t)+z_2t$ for $t \in [0,1]$. \cite{bowman_generating_2016} find that this allows to generate coherent, novel sequences that interpolate between known sequences. Examining homotopies allows to get a sense of how the latent space is organised and how the model deforms a sentence to transition between the two sentences at the opposite ends of the line.\\

\subsection{Conditional Variational Autoencoders}
While a variational autoencoder can successfully generate plausible outputs for both continuous and discrete tasks, the generation process is unguided as samples are drawn from the prior $p(\mathbf{z}) = \mathcal{N}(\boldsymbol{0}, \mathbf{I})$ without any control over the properties of the output. Conditional generative models address this by incorporating additional contextual information to steer generation toward a desired outcome or, in other words, to achieve \textit{controlled} generation. The Conditional Variational Autoencoder (CVAE), introduced by Sohn et al. \cite{sohn_learning_2015}, extends the VAE framework to allow for structured output prediction: by conditioning both the encoder and the decoder on an observed variable $\mathbf{x}$, the model learns a structured, semantically rich latent space. As a result, samples from a given region of the code are decoded into outputs that feature specific properties.\\

A CVAE is trained to maximise the conditional log-likelihood of the data $\log p_\theta(\mathbf{y}|\mathbf{x})$, where $\mathbf{x}$ denotes the input variables and $\mathbf{y}$ denotes the output variables. The variational lower bound is modified as follows:
\begin{equation}
\log p_\theta(\mathbf{y}|\mathbf{x}) \geq -D_{\text{KL}}\left(q_\phi(\mathbf{z}|\mathbf{x}, \mathbf{y}) \| p_\theta(\mathbf{z}|\mathbf{x})\right) + \mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x},\mathbf{y})}\left[\log p_\theta(\mathbf{y}|\mathbf{x}, \mathbf{z})\right],
\end{equation}

where $\mathbf{z}$ denotes the latent variable. The empirical lower bound used to train the model is:
\begin{equation}
\tilde{\mathcal{L}}_{\text{CVAE}}(\mathbf{x}, \mathbf{y}; \theta, \phi) = -D_{\text{KL}}\left(q_\phi(\mathbf{z}|\mathbf{x}, \mathbf{y}) \| p_\theta(\mathbf{z}|\mathbf{x})\right) + \frac{1}{L}\sum_{l=1}^{L} \log p_\theta(\mathbf{y}|\mathbf{x}, \mathbf{z}^{(l)})
\end{equation}
where $\mathbf{z}^{(l)} = g_\phi(\mathbf{x}, \mathbf{y}, \boldsymbol{\epsilon}^{(l)})$, $\boldsymbol{\epsilon}^{(l)} \sim \mathcal{N}(\boldsymbol{0}, \mathbf{I})$ and $L$ is the number of samples.\\

Compared to the vanilla VAE, the CVAE introduces two key differences. First, the KL divergence term now measures the divergence between the recognition network $q_\phi(\mathbf{z}|\mathbf{x}, \mathbf{y})$, which has access to both the input and the output during training and the prior network $p_\theta(\mathbf{z}|\mathbf{x})$, which conditions only on the input. Second, the prior is no longer a fixed standard Gaussian, but a learned distribution conditioned on $\mathbf{x}$, allowing the model to adapt the latent space to the input context. At inference time, the recognition network is discarded and samples are drawn from the prior network $p_\theta(\mathbf{z}|\mathbf{x})$, which steers generation toward outputs consistent with the observed input.\\

%where $\alpha$ balances the two objectives. When $\alpha = 1$, the hybrid objective recovers the CVAE objective; when $\alpha = 0$, the model reduces to a GSNN without the recognition network.

Zhang et al. \cite{zhang_variational_2016} extend the CVAE framework to discrete sequential data by replacing the visual components with RNN encoder-decoders, applying it to neural machine translation and demonstrating improvements over vanilla Seq2Seq models. In both cases, the CVAE learns a structured latent space conditioned on the input, enabling the model to generate diverse outputs with specific properties at inference time.\\

\subsection{Variational Autoencoders with Contrastive Learning}
Another approach to learning a structured, semantically rich latent space is to augment the VAE training objective with a contrastive loss. Contrastive representation learning aims to learn an embedding space in which the distance between samples is a proxy for semantic similarity by pulling semantically similar data together and pushing semantically different data apart. Depending on how similarity is defined, contrastive learning can be applied in both supervised and unsupervised settings.\\

Triplet loss was originally proposed in \cite{schroff_facenet_2015} in the context of supervised learning and uses one positive and one negative per \textit{anchor}. Given one anchor input $\textbf{x}_{i}$, one positive sample $\textbf{x}_{i}^{+}$ and one negative sample $\textbf{x}_{i}^{-}$ are selected, such that $\textbf{x}_i$ and $\textbf{x}_{i}^{+}$ belong to the same class $\textbf{x}_{i}$ and $\textbf{x}_{i}^{-}$ belong to different classes. Triplet loss learns to minimise the distance between the anchor $\textbf{x}_{i}$ and the positive sample $\textbf{x}_{i}^{+}$ and to maximise the distance between the anchor $\textbf{x}_{i}$ and negative sample $\textbf{x}_{i}^{-}$ via the following equation:
\begin{equation}
\mathcal{L}_{\text{Triplet}}(\mathbf{x}, \mathbf{x}^{+}, \mathbf{x}^{-}) = \sum_{\mathbf{x} \in \mathcal{X}} \max\left(0, \|f(\mathbf{x}) - f(\mathbf{x}^{+})\|_{2}^{2} - \|f(\mathbf{x}) - f(\mathbf{x}^{-})\|_{2}^{2} + \epsilon\right)
\end{equation}
 
where $\epsilon$, also called \textit{margin}, is a hypermarater that controls the minimum gap that must exist between the distance to the positive and the distance to the negative.\\

Drawing on the work in \cite{gutmann_noise-contrastive_nodate}, Oord et al. \cite{oord_representation_2018} introduce the InfoNCE objective. Given a set of samples $\{x_i\}_{i=1}^N$, where only one, $x^+$, is a positive sample and the remaining are negative samples, InfoNCE minimises the following loss:
\begin{equation}
    \mathcal{L}_{\text{InfoNCE}} = -\mathbb{E}\left[\log \frac{f(\mathbf{x}^{+}, \mathbf{c})}{\sum_{\mathbf{x}_i \in X} f(\mathbf{x}_i, \mathbf{c})}\right]
\end{equation}

where $f(\mathbf{x}, \mathbf{c})$ denotes a scoring function measuring the compatibility between sample $\mathbf{x}$ and context representation $\mathbf{c}$.\\

Khosla et al. \cite{khosla_supervised_2021} suggest another version of contrastive learning for the supervised setting, SupCon. SupCon can be seen as a generalization of the triplet loss from \cite{schroff_facenet_2015} as it utilises multiple positive and negative samples:
\begin{equation}
\mathcal{L}_{\mathrm{SupCon}}
=
\sum_{i \in I}
-\frac{1}{|P(i)|}
\sum_{p \in P(i)}
\log
\frac{
\exp(\mathbf{z}_i \cdot \mathbf{z}_p / \tau)
}{
\sum_{a \in A(i)}
\exp(\mathbf{z}_i \cdot \mathbf{z}_a / \tau)
}
\end{equation}

where $I$ is the size of the batch, $P(i)$ denotes the set of indices corresponding to positive samples for anchor $i$, $A(i)$ denotes the set of all candidate samples excluding the anchor itself, $\tau$ is a temperature hyperparameter and $\mathbf{z}_i$ represents the projected embedding of sample $i$.\\

\subsection{Adversarial Variational Autoencoders}

Adversarial approaches to variational autoencoders aim to improve generative quality by introducing discriminator-based training signals. In the context of text generation, Bowman et al. \cite{bowman_generating_2016} evaluate VAE-generated sentences by training a classifier to distinguish between real and generated samples. They define an evaluation metric that is adversarial in spirit and idenfity fully adversarial training of text VAEs as a direction for future research. The approach in \cite{bowman_generating_2016} is motivated by the discrete nature of text, where the RNN decoder produces sequences of discrete tokens, making the sampling process non-differentiable and preventing direct backpropagation of adversarial objectives through the generator.\\

Hu et al. \cite{hu_toward_2018} propose a framework for controllable text generation that combines variational autoencoders with adversarially trained discriminators. The model addresses two key challenges: learning meaningful latent representations for text and enabling control over specific semantic attributes. To this end, the authors extend the wake-sleep algorithm from Hinton et al. \cite{hinton_wake-sleep_1995}, where the generative model produces synthetic samples or \textit{dreams} that are used to adversarially train the auxiliary discriminator, while the inference model is updated using reconstruction-based objectives. In addition, attribute discriminators are introduced to enforce that generated sentences exhibit desired properties, while regularization terms encourage disentanglement between latent variables and controllable attributes. This hybrid training procedure results in improved controllability and more meaningful latent representations, although performance remains limited to relatively short and constrained sentence generation. The objectives of the generator $G$ and of the discriminator $D$ are respectively defined as follows:
\begin{align}
\mathcal{L}_G &= \mathcal{L}_{\text{VAE}} + \lambda_c \mathcal{L}_{\text{Attr},c} + \lambda_z \mathcal{L}_{\text{Attr},z} \\
\mathcal{L}_D &= \mathcal{L}_s + \lambda_u \mathcal{L}_u
\end{align}

where $\lambda_c$, $\lambda_z$ and $\lambda_u$ are all balancing parameters. $\mathcal{L}_{VAE}$ denotes the standard objective of a vanilla variational encoder. $\mathcal{L}_{\text{Attr},c}$ it measures how well generated text matches the desired attribute using a classifier. $\mathcal{L}_{\text{Attr},z}$ incentives the model to learn disentangled representations. $\mathcal{L}_s$ is the supervised classification loss on labeled real data. $\mathcal{L}_u$ is the unsupervised loss on synthesised (generated) samples used to regularise the discriminator.
